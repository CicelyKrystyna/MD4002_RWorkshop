An Introduction to R#

R is a language and environment for statistical computing and graphics. You can read more about it here.

In this Notebook we will point out some useful basics of R to help you get started. If you want to learn more then there are a variety of resourses listed in the references section.

We will be assuming you are following along using RStudio, but you can also interact with the code by clicking the rocket launch button to open this is Google Colab.

To get started (having downloaded R and RStudio) open up RStudio and open a new R Script (e.g. File->New File->R Script). Letâ€™s save this script as RBasics.R and get started.

Getting to know R/RStudio#

When you are working in R Studio you should see that the window is split into four panels. Commonly this will be:

  • top left - any current script(s) (e.g. our RBasics.R script)

  • bottom left - a console for experimental commands and any R output

  • top right - the working environment with any saved variables or data structures

  • bottom right - extras such as a files tab and R Help

The Console#

in If you want to test out a line of code (commands) you can run it in the console before copying it to your script. Some useful features:

  • You can use your keyboards up/down arrows to scroll back through previous commands.

  • You can use ^R to search your command history.

  • You can clear the console using ^L.

Tip

You can also use the console to access Râ€™s help guides (which will appear in the bottom right panel).

You can search Râ€™s documentation for a keyword by typing ??keyword. You can get help on a particular function or package by typing ?function.

??mean
# asking R to provide help on the function mean
?mean

In RStudio you can create (collapsible) sections to keep your code organised. To create a section in R Studio go to Code->Insert Section, a pop up box will appear for you to name your section. Clear R environment#

A common line you will often see in the Preamble of R codes is one which will clear the current workspace/environment.

# Clear everything from the environment
rm(list = ls())

Caution

This will delete/unassign all of the varaibles and data structures you have created so only use it if you wont lose something from the environment you will need.

Objects in R#

Objects in R are essentially containers. They may contain single peices of data (e.g. a variable) or be more complex structures of data (e.g. lists and data frames). Objects are created by assigning the data to that object using <-.

Note

There are actually no less than five different assingment operators as per this documentation. However, at a basic level using <- will serve you well.

Here are some examples:

# assign the value 16 to the object my_variable
my_variable<-16
# assign a set of values {1,2,3} to the object my_set
my_set<-c(1,2,3)
# assign a set of values {1,2,3} to the object my_list
my_list<-list(1,2,3)
# assign a set of sets {1,2,3} and {"A", "B", "C", "D"} to the object my_listofsets
my_listofsets<-list(c(1,2,3),c("A","B","C","D"))
my_listofsets<-list(my_set1=c(1,2,3),my_set2=c("A","B","C","D"))
# assign a table of data to the object (dataframe) df
my_dataframe<-data_frame(my_column1=c(1,2,3), my_column2=c("yes","no","maybe"))

Note

A list can contain sets of different length. In a dataframe every column must have the same length.

In R Studio you will see all of the objects listed in the environment panel. Alternatively you can get R to list all the currently assigned objects using ls().

ls()

  2. 'my_list'
  3. 'my_listofsets'
  4. 'my_set'
  5. 'my_variable'

We saw above that rm(list = ls()) will remove (and de-assign) all of the objects in the current session. You can remove/deassign individual objects using, for example, rm(my_variable).
